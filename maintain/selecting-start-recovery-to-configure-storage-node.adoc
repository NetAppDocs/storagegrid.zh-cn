= 选择 Start Recovery 以配置 StorageGRID 存储节点
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
更换存储节点后，您必须在网格管理器中选择启动恢复，以将新节点配置为故障节点的替代节点。

.开始之前
* 您已使用登录到网格管理器link:../admin/web-browser-requirements.html["支持的 Web 浏览器"]。
* 您拥有link:../admin/admin-group-permissions.html["维护或root访问权限"]。
* 您具有配置密码短语。
* 您已部署并配置替代节点。
* 您知道已进行过数据检索的任何修复作业的开始日期。


.关于此任务
如果存储节点作为容器安装在 Linux 主机上，则只有在满足以下条件之一时，才必须执行此步骤：

* 您必须使用 `--force`标志导入节点、否则会发出此命令 `storagegrid node force-recovery _node-name_`
* 您必须执行完整节点重新安装，或者需要还原 /var/local 。


.步骤
. 登录到主管理节点。如果主管理节点不可用，请使用非主管理节点（如果存在）。
. 在网格管理器中，选择*Maintenance* > *Tasks* > *Node recovery*。
. 在 Pending Nodes 列表中选择要恢复的网格节点。
+
节点发生故障后会显示在列表中、但您无法选择某个节点、直到它重新安装并准备好进行恢复为止。

. 输入 *Provisioning passphrase*。
. 单击*开始恢复*。
. 通过选择*维护* > *任务* > *节点恢复*并查看恢复进度来监控恢复进度。
+
** 如果网格节点达到"等待手动步骤"阶段，则必须link:../maintain/remounting-and-reformatting-storage-volumes-manual-steps.html["手动重新挂载并重新格式化存储卷"]。完成手动步骤后，请继续link:../maintain/restoring-object-data-to-storage-volume.html["将对象数据还原至卷"]。
** 如果网格节点恢复已从*进行中*过渡到*已恢复*，而无需经过"等待手动步骤"阶段，则无需执行进一步操作。
+
[NOTE]
====
在恢复过程中的任何时候，您都可以选择*重置*以开始新的恢复。将显示一个对话框，指示如果重置过程，节点将保持不确定状态。

如果要在重置过程后重试恢复，则必须将设备节点恢复到预安装状态，如下所示：

*** * VMware * ：删除已部署的虚拟网格节点。然后，当您准备好重新启动恢复时，重新部署节点。
*** *Linux*：通过在Linux主机上运行以下命令来重新启动节点： `storagegrid node force-recovery _node-name_`


====
