= 检查 StorageGRID 中的数据修复作业
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
在退役或扩展网格节点之前，您必须确认没有处于活动状态的数据修复作业。如果任何修复失败，则必须重新启动它们并让它们完成，然后再执行退役程序。

* 参考link:../expand/index.html["扩展类型"]有关如何扩展网格节点的详细信息。
* 使用 Grid Manager 监控正在进行的恢复过程并显示恢复历史记录。请参阅以下步骤：
+
** link:../maintain/restoring-object-data-to-storage-volume-for-appliance.html["将对象数据还原到设备的存储卷"]
** link:../maintain/restoring-object-data-to-storage-volume-where-system-drive-is-intact.html["将对象数据还原到系统驱动器完好无损的存储卷"]
** link:../maintain/restoring-object-data-to-storage-volume.html["将对象数据还原到存储卷(系统驱动器故障)"]
